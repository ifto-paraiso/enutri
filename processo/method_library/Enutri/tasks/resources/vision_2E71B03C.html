Artefato: Visão
Este artefato contém a definição da visão dos Stakeholders a respeito do produto a ser desenvolvido, especificada em termos das principais características e necessidades dos Stakeholders. Contém um esboço dos principais requisitos vislumbrados para o sistema.
Domínios: Requisitos
Tipos de Produto de Trabalho: Conceito
Objetivo

Este artefato fornece uma base de alto nível, às vezes contratual, para os requisitos técnicos mais detalhados que são visíveis para os Stakeholders. Captura a essência do sistema descrevendo as restrições de design e os requisitos de alto nível que dão ao leitor uma visão geral do sistema sob uma perspectiva de requisitos comportamentais. Serve como entrada para o processo de aprovação do projeto, comunica os "o que e porque" fundamentais para o projeto, e fornece um plano no qual todas as decisões futuras devem ser validadas.

Descrição
Descrição Principal

Este artefato fornece uma visão completa do sistema de software em desenvolvimento e dá suporte ao contrato entre o cliente e a organização de desenvolvimento. Cada projeto necessita de uma fonte para capturar as expectativas dos Stakeholders.

Este artefato é escrito a partir da perspectiva dos clientes, focando nas características essenciais do sistema e nos níveis aceitáveis de qualidade. O artefato deve incluir uma descrição de quais Características serão incluídas, bem como as que foram consideradas, mas não incluídas.

Adaptação
Impacto de não terSe este artefato não for usado, existira um alto risco de que os Stakeholders e a equipe de desenvolvimento tenham diferentes expectativas. Isto pode conduzir ao cancelamento do projeto.
Opções de RepresentaçãoCustomize este artefato para as necessidades do seu projeto. Normalmente é uma boa prática manter este artefato resumido de forma que você possa liberá-lo aos Stakeholders o mais rápido possível, e torna-lo fácil de ser lido e compreendido pelos Stakeholders. Você pode conseguir isto, incluindo somente as mais importantes características e solicitações dos Stakeholders evitando os detalhes dos requisitos. Você pode descrever os detalhes em outros artefatos para requisitos.
