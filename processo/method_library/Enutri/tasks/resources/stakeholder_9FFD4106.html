Papel: Stakeholder
Este papel representa grupos de interessados cujas necessidades devem ser satisfeitas pelo projeto. É um papel que pode ser executado por qualquer um que seja (ou potencialmente possa ser) materialmente afetado pelo resultado do projeto.
Conjuntos de Papéis: Papéis no OpenUP
Relacionamentos
Desempenha Adicionalmente
  • Avaliar os Resultados
  • Criar os Casos de Teste
  • Definir a Visão
  • Descrever a Arquitetura
  • Detalhar os Requisitos
  • Encontrar e Descrever os Requisitos
  • Gerenciar a Iteração
  • Implementar a Solução
  • Implementar os Scripts de Teste
  • Planejar a Iteração
  • Planejar o Projeto
  • Projetar a Solução
